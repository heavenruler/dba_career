## Database Server 数据库服务
Oracle AI 数据库至少包含一个数据库实例和一个数据库。数据库实例由内存和进程组成，用于管理多租户容器数据库（CDB）。CDB 由物理存储结构（数据文件）和逻辑数据结构（模式和模式对象）组成。

此处展示的是单实例数据库架构，其中数据库与数据库实例之间是一对一的关系。同一台服务器上可以存在多个单实例数据库，这种配置适用于在同一台机器上运行不同版本的 Oracle AI 数据库。

Oracle Real Application Clusters（Oracle RAC）数据库架构由运行在不同服务器上的多个实例和一个共享数据库组成。有关 Oracle RAC 架构的更多信息，请参阅引用链接。

Oracle Net 监听器是数据库服务器进程，接收来自客户端进程的传入连接，建立与数据库实例的连接，然后将客户端连接移交以直接与服务器进程通信，从而满足客户端请求。单实例数据库的监听器可以在数据库服务器本地运行，也可以在远程运行。

您可以使用多种工具来管理数据库，包括：
- Oracle Data Pump
- Oracle Enterprise Manager
- Oracle AI Database AutoUpgrade
- Database Configuration Assistant (DBCA)
- Oracle Recovery Manager (RMAN)
- Oracle Server Control Utility (SRVCTL)
- SQL Developer
- SQL*Loader
- SQL*Plus

## Database Instance 数据库实例
在 Oracle AI 数据库中，数据库实例是一组进程和内存结构的集合，用于管理数据库文件。主要的内存结构包括系统全局区（SGA）和程序全局区（PGA）。Managed Global Area（托管全局区，MGA）是一个内存框架，能够在一组受信任的 Oracle 进程中共享和协调内存。后台进程对数据库文件进行操作，并使用内存结构来完成其工作。数据库实例仅存在于内存中。

数据库实例还具有服务器进程，用于代表客户端程序处理与数据库的连接，并为客户端程序执行工作。例如，这些服务器进程解析并运行 SQL 语句、检索结果并返回给客户端进程。这类服务器进程也称为前台进程。借助多租户架构，一个数据库实例与单一的多租户容器数据库（CDB）关联。所有插入到 CDB 中的可插拔数据库（PDBs）（无论是直接插入还是通过应用程序容器插入）共享一组后台进程和 SGA。

### SGA 系统全局区
系统全局区（SGA）是包含单个 Oracle AI 数据库实例的数据和控制信息的内存区域。所有服务器进程和后台进程共享 SGA。启动数据库实例时，会显示分配给 SGA 的内存量。SGA 包含以下数据结构：

- 共享池：缓存各种可供多个用户共享的结构，例如已解析的 SQL、PL/SQL 代码、系统参数和数据字典信息。数据库中发生的几乎所有操作都与共享池有关。
- （可选）向量池：要允许创建和维护向量索引，必须启用存储在 SGA 中的名为“向量池”的内存区域。向量池是 SGA 中分配的内存，用于存储分层可导航小世界（HNSW）向量索引及其所有关联元数据。它还用于加速倒排文件平面（IVF）索引的创建，以及对具有 IVF 索引的基表执行 DML 操作。
- （可选）闪回缓冲区：与闪回数据库配合使用，以便将数据回溯到过去某个时间点来纠正问题。启用闪回数据库后，恢复写入器进程（RVWR）会定期将修改过的数据块从缓冲区缓存复制到闪回缓冲区，并将闪回数据库数据从闪回缓冲区按顺序写入闪回数据库日志，这些日志会循环重用。
- 数据库缓冲区缓存：存储从数据文件中读取的数据块副本的内存区域。缓冲区是主内存中的一个地址，缓冲区管理器会将当前或最近使用的数据块临时缓存到该地址中，所有并发连接到数据库实例的用户共享对缓冲区缓存的访问权限。
- （可选）True Cache：Oracle AI 数据库的内存缓存，支持 SQL 和键值（对象或 JSON）缓存，具有一致性和自动管理功能。可以将常用表持久化到 True Cache 的数据库缓冲区缓存或主数据库（或两者）。
- （可选）数据库智能闪存缓存：运行在 Solaris 或 Oracle Linux 上的数据库的数据库缓冲区缓存的内存扩展，为数据库块提供二级缓存，常用于读取密集型 OLTP 和数据仓库环境。
- 重做日志缓冲区：SGA 中的循环缓冲区，用于保存数据库更改信息（重做条目）。
- （可选）大型池：用于分配比共享池更大的内存，为共享服务器的 UGA、Oracle XA 接口、并行处理消息缓冲区、RMAN I/O 工作进程的缓冲区以及延迟插入提供大容量内存。
- fixed SGA：内部管理区域，包含有关数据库和数据库实例状态的一般信息以及进程之间通信的信息。

### In-Memory Area（可选）
内存区域允许将对象（表、分区等）以列式格式存储在内存中，以加速扫描、连接和聚合操作，适用于分析型查询。内存区域与行存储同时存在于 SGA 中，数据库会透明地将 OLTP 查询发送到缓冲区缓存，并将分析查询发送到 In-Memory 列式存储。In-Memory 区域的主要组件包括 IMCU（内存压缩单元）、SMU（快照元数据单元）、IMEU（内存表达式单元）和 ESS（表达式统计信息存储库）。In-Memory 区域大小由 INMEMORY_SIZE 参数控制，最小为 100MB，且可以动态增大但不能动态缩小。

## PGA 程序全局区
程序全局区（PGA）是数据库实例内的非共享内存区域，包含仅供服务器或后台进程使用的数据和控制信息。Oracle AI 数据库为每个启动的服务器和后台进程创建一个 PGA（专用服务器环境）。当关联的服务器或后台进程终止时，Oracle AI 数据库会释放该 PGA。

在专用服务器会话中，PGA 由以下组件构成：
- SQL 工作区：包括排序区（用于 ORDER BY、GROUP BY）、哈希区（用于哈希连接）以及位图合并区（用于位图索引扫描合并）。
- 用户全局区（UGA）：存储会话变量（如登录信息）和会话所需的其他信息，以及管理 OLAP 数据页的 OLAP 池。
- 私有 SQL 区域：保存已解析 SQL 语句的信息以及其他会话特定的处理信息，如绑定变量值、查询执行状态信息和查询执行工作区等。游标是指向私有 SQL 区域中特定区域的名称或句柄。

在共享服务器环境中，多个客户端用户共享服务器进程。UGA 会移至大池，而 PGA 则仅保留 SQL 工作区和私有 SQL 区域。

## 后台进程 Background processes
后台进程是数据库实例的一部分，负责执行维护任务，以确保数据库正常运行并最大限度地提高多用户性能。每个后台进程执行的任务各不相同，但彼此协同工作。Oracle AI 数据库会在启动数据库实例时自动创建后台进程，具体存在的后台进程取决于使用的数据库功能。启动数据库实例时，必需的后台进程会自动启动，可选后台进程可在需要时稍后启动。

必需的后台进程包括：进程监视器 (PMON)、进程管理器 (PMAN)、监听器注册 (LREG)、系统监视器 (SMON)、数据库写入器 (DBWn)、检查点 (CKPT)、可管理性监视器 (MMON)、精简版可管理性监视器 (MMNL)、恢复器 (RECO) 和日志写入进程 (LGWR)。

常见的可选后台进程包括：归档程序 (ARCn)、作业队列协调程序 (CJQ0)、恢复写入程序 (RVWR)、闪回数据归档程序 (FBDA) 和空间管理协调程序 (SMCO)。工作进程（如 Dnnn、Snnn）是后台进程，它们代表其他进程执行工作。

## Shared Pool 共享池
共享池是 SGA 的一个组成部分，负责缓存各种类型的程序数据，例如已解析的 SQL、PL/SQL 代码、系统参数和数据字典信息。共享池由以下子组件构成：

- Library Cache（库缓存）：用于存储可执行的 SQL 和 PL/SQL 代码，包含共享的 SQL 和 PL/SQL 区域以及控制结构（如锁和库缓存句柄）。当执行 SQL 语句时，数据库会尝试重用之前执行过的代码（软解析/库缓存命中）；否则需进行硬解析。
- reserved pool（预留池）：共享池中的一个内存区域，可用于分配大块连续内存，避免因内存碎片导致的大对象加载失败。
- data dictionary cache（数据字典缓存）：存储数据库对象的信息（行缓存形式）。
- server result cache（服务器结果缓存）：包含 SQL 查询结果缓存和 PL/SQL 函数结果缓存。
- 其他辅助组件：入队、锁存器、信息生命周期管理（ILM）位图、活动会话历史（ASH）缓冲区等。

## Large Pool 大池
大池是 SGA 中的一个可选内存区域，可为以下用途提供专用大内存分配：
- 在共享服务器环境和 Oracle XA 接口中存放 UGA（在专用服务器环境中，UGA 存在 PGA）。
- I/O 缓冲区区域，包括 I/O 服务器进程、并行查询的消息缓冲区、RMAN I/O 工作进程的缓冲区以及高级队列内存表存储。
- 快速摄取功能使用的延迟插入池（用于 MEMOPTIMIZE FOR WRITE 表的高频单行插入）。
- 并行执行消息池（px msg pool）用于并行查询执行进程之间发送消息。

大池与共享池的预留空间不同：大池没有 LRU 列表，内存一旦分配，只有在使用完毕后才能释放。

在共享服务器环境中，用户请求的处理流程大致如下：
1. 客户端进程向数据库实例发送请求，调度程序进程（Dnnn）接收请求。
2. 调度程序将请求放入大池的请求队列中。
3. 下一个可用的共享服务器进程（Snnn）会处理该请求，访问库缓存验证数据项，并访问缓冲区缓存检索数据（若缓存未命中则访问磁盘）。
4. 一个请求的不同数据库调用可能由不同的共享服务器进程处理，因此 UGA 必须是共享内存区域，供所有共享服务器进程访问。
5. 请求完成后，共享服务器进程将响应放入位于大池中的响应队列。
6. 响应队列将响应发送给调度程序，调度程序再将完成的请求返回给相应的客户端进程。

## Vector Pool 向量池
为允许创建向量索引，Oracle AI Database 引入了存储在 SGA 中的新内存区域——向量池（Vector Pool）。向量池用于存储 HNSW 向量索引及其相关元数据，也用于加快 IVF 索引的创建与维护，并在执行带有 IVF 索引的基表 DML 操作时提升性能。IVF 向量存储在向量池中以提高创建和维护性能；如果向量池空间不足，质心向量将被缓存到大池；如果大池也不足，则质心向量不会被缓存到内存中。此规则适用于未分区和已分区的本地及全局 IVF 索引。

可以通过 VECTOR_MEMORY_SIZE 初始化参数在本地环境中调整 Vector Pool 大小。该参数可在不同层级动态修改：
- 在 CDB 层面，VECTOR_MEMORY_SIZE 指定向量池的当前大小；如果已有向量在使用，则减小参数值将失败。
- 在 PDB 层面，VECTOR_MEMORY_SIZE 指定 PDB 允许的最大向量池使用量；即使当前向量使用量超过新的配额，也允许降低此参数值。

更改参数值的方法包括：
- 编辑初始化参数文件（通常在下一次启动实例时生效）。
- 使用 ALTER SYSTEM SET ... SCOPE=SPFILE 更新服务器参数文件。
- 使用 ALTER SYSTEM RESET 清除初始化参数值并设置回默认值。

示例 SQL 会话：
```sql
SQL> show con_name
CON_NAME
------------------------------
JIEKEXUPDB1
SQL> show user
USER is "SYS"
SQL> show parameter vector_memory_size
NAME TYPE VALUE
------------------ ----------- -----
vector_memory_size big integer 500M
SQL> SELECT ISPDB_MODIFIABLE
2 FROM V$SYSTEM_PARAMETER
3* WHERE NAME='vector_memory_size';
ISPDB_MODIFIABLE
___________________
TRUE
SQL> ALTER SYSTEM SET vector_memory_size=1G SCOPE=BOTH;
System altered.

SQL> show parameter vector_memory_size
NAME TYPE VALUE
------------------- ----------- -------
vector_memory_size big integer 1G
SQL>
```

如果 VECTOR_MEMORY_SIZE 设置为 1（例如 1G），且在 CDB 初始化时 sga_target 大于 0，则 HNSW 索引创建会自动增大向量内存池以满足新索引需求。PDB 的 VECTOR_MEMORY_SIZE 最大值限制为 PDB sga_target 的 70%。删除 HNSW 索引时，向量内存池会相应缩小。在自动增长情况下，PDB 的 VECTOR_MEMORY_SIZE 值将默认为 0，且不能使用 ALTER SYSTEM 命令修改。自动增长导致的向量池大小变化不会保存到 spfile 中，因此数据库重启时向量池大小会被重置。可通过视图 V$VECTOR_MEMORY_POOL 监控向量池。

## 缓冲区缓存 buffer cache
数据库缓冲区缓存（缓冲区缓存）是 SGA 中的内存区域，存储从数据文件中读取的数据块副本。缓冲区由缓冲区管理器管理，目标是优化物理 I/O 并将频繁访问的数据块保留在内存中。缓存命中时可直接从内存读取数据，未命中时需要将磁盘上的数据块复制到缓存后访问。

缓冲区缓存由以下区域组成：
- 默认池：数据块通常缓存的位置（默认块大小 8 KB）。
- （可选）保留池：用于放置预计会被频繁访问且因空间不足而从默认池中移除的数据块，以避免 I/O。
- （可选）回收池：用于存放不常用的数据块，防止占用缓存空间。
- （可选）非默认缓冲池：适用于 2 KB、4 KB、16 KB、32 KB 等非标准块大小的表空间，每种块大小都有对应的缓冲池。
- （可选）数据库智能闪存缓存（闪存缓存）：使用闪存设备增加缓冲区缓存容量，优先在缓冲区缓存查找，其次在闪存缓存查找，最后才从磁盘读取。RAC 环境中必须在所有实例上统一配置闪存缓存或不配置。
- 最近最少使用（LRU）列表：包含指向脏缓冲区和非脏缓冲区的指针，LRU 列表分为热端和冷端。为保证并发性，数据库实际上使用多个 LRU 列表。
- 检查点队列：脏缓冲区的列表，按重做块地址（RBA）顺序排列。
- （可选）闪存缓冲区包含默认闪存 LRU 链和保留闪存 LRU 链。对象的 FLASH_CACHE 属性决定缓冲区头在闪存缓存或内存中的位置。系统不会将脏缓冲区放入闪存缓存。

## IM 内存区域（In-Memory Area）
In-Memory 区是 SGA 的可选组件，包含内存列式存储（IM 列式存储），用于将表和分区以列式格式保存在内存中，从而显著加速分析查询。IM 列式存储允许与行格式并存，数据库会根据查询类型选择最优存储区域。IM 列式存储使用 IMCU（压缩只读存储单元）和 SMU（快照元数据单元）管理数据和元数据；IMEU（内存表达式单元）用于存储物化 IM 表达式和用户定义虚拟列；ESS（表达式统计信息存储库）用于存储表达式求值统计信息并驻留于 SGA。

内存区域分为两个池：1MB 的列式数据池（存储 IMCU 和 IMEU）和 64KB 的元数据池（存储 SMU 元数据）。In-Memory 区大小由 INMEMORY_SIZE 参数控制，默认 0，最小 100MB；从 Oracle Database 12.2 开始，可以通过 ALTER SYSTEM 动态增加 In-Memory 区大小，但无法动态减小。

首次访问内存表数据或数据库启动时，会在内存列存储中为内存表分配 IMCU。通过磁盘格式转换为内存列格式创建表的内存副本，实例重启时此副本需重新创建。表可以部分转换，查询会在部分内存版本和磁盘之间混合读取（内存混合扫描），无需等待整个表转换完成。

自动内存功能（INMEMORY_AUTOMATIC_LEVEL）可自动启用、填充、逐出和重新压缩段。相关后台进程包括内存协调器进程（IMCO）、空间管理协调器进程（SMCO）和空间管理工作进程（Wnnn），负责列式数据的后台填充与重新填充。服务器进程负责响应查询和 DML，扫描列式数据并更新 SMU 元数据。内存快速启动功能可将列式数据以压缩格式写回表空间以加快数据库启动速度（不适用于 IMEU）。内存深度向量化（INMEMORY_DEEP_VECTORIZATION）通过 SIMD 技术对 SQL 运算符进行向量化，默认启用，可通过初始化参数关闭。
