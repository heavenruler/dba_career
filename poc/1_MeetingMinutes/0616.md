## 5. 非維運端的其他 DBA 層面議題

> 目的：除維運（部署 / 監控 / 備份）外，分散式資料庫導入會牽動其他 DBA 工作層面；本項列舉討論範圍，於會中由 DBA 與 application owner 共同點評優先序與責任分工。

### 5.1 議題清單

| 層面 | 議題 | 預期討論結論 |
|---|---|---|
| **Schema / DDL** | 分片鍵（shard key）選擇對 join / range query 的影響、二級索引設計成本、DDL online change 範圍與停機窗口 | 設計準則 + 範例情境 |
| **交易模式** | 短交易 vs 長交易、批次處理切片大小、retry 策略納入應用層的程式碼修改範圍 | 與 application owner 對齊的 coding guideline |
| **資料模型** | 熱點分散（hot row / hot range）、序列 / UUID / hash key 之間取捨、跨分片 foreign key 行為 | 模型轉換 checklist |
| **連線層 / 連線池** | HAProxy / pgbouncer / 應用內 connection pool 三層架構選型、連線數上限、idle 回收 | 標準連線層拓撲圖 |
| **備份 / 還原 / 點時還原（PITR）** | 邏輯備份 vs 物理備份的取捨、PITR 精度（秒 / 分）、跨區域備份保留期 | 備份策略 SLA |
| **資料遷移** | dual-write / change data capture（CDC）/ cutover window 三種策略適用情境、回滾機制 | 遷移 playbook + dry-run 流程 |
| **DR / failover** | RTO / RPO 目標、自動 failover 與人工介入分界、跨區 failover drill 頻率 | DR runbook + drill 排程 |
| **存取控制 / 稽核** | RBAC 對齊既有 AD / LDAP、查詢稽核 log 保存期、敏感資料遮罩 | 安全控管 SOP |
| **效能調校** | 查詢計畫穩定性、統計資訊更新、執行計畫快取與 plan baseline | 調校 SOP（與應用團隊共同）|
| **觀測 / SLO** | metrics（Prometheus）/ trace / log 三層整合、SLO 指標（吞吐 / 延遲 / 錯誤率）告警閾值 | 觀測 baseline + SLO 草案 |
| **開發 / 測試環境** | sandbox / staging / 生產三層一致性、CI 測試對接分散式 DB 的成本、developer self-service 程度 | 環境分層策略 |
| **on-call / training** | DBA on-call 表、應用團隊基本訓練、第二線升級流程、 vendor 升級窗口 | on-call rota + training plan |
| **資料生命週期** | 歷史資料封存（archive）、TTL / 自動 purge、跨分片 vacuum / compaction 行為 | 生命週期 SOP |
| **多租戶 / multi-tenancy** | 單 cluster 多 schema / 多 cluster / 多 namespace 取捨、cross-tenant 資源隔離 | 租戶模型決策 |
