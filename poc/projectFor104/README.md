# [分散式資料庫架構 PoC](https://104corp.atlassian.net/browse/ITDBA-3596)

## 1. PoC 目標

本 PoC 用於驗證分散式資料庫架構是否可滿足 104Corp 既有業務系統需求。

北極星目標：分散式資料庫部署不應因任何因素（例如停機維護）導致服務中止或暫停。

- 可用性：單點故障時可持續提供服務
- 擴充性：可支援資料量與流量成長
- 一致性：確認交易、複寫與故障切換行為
- 維運性：備援、監控、告警、備份與還原流程可操作
- 成本可行性：評估導入與營運成本是否合理

### 2.5 維運規劃

- 備份策略設計 & 規劃

### 2.6 風險與結論

## 3. 執行計劃

### Phase 1. 需求確認

- 盤點業務場景與痛點
- 定義 RTO / RPO、吞吐量、延遲、併發需求
- 確認成功判定條件與 PoC 範圍

### Phase 2. 架構選型

- 比較候選方案
- 評估優缺點、風險與導入複雜度
- 確認與現況整合方式及選型理由

### Phase 3. 架構與環境設計

- 繪製邏輯架構圖與實體部署圖
- 規劃節點角色、資料路徑與故障切換流程
- 確認主機、網路與容量需求

### Phase 4. 測試驗證

- 執行功能測試
- 執行 TPC-C 壓力測試與穩定性測試
- 執行故障注入、切換與備份還原測試
- 依驗收門檻彙整結果

### Phase 5. 維運與結案

- 完成備份策略設計與規劃
- 盤點風險、限制與待解議題
- 產出測試結果摘要與導入建議
