# Chaos C4 — IDC leader die

## 注入方式

停止 IDC 端 leader node（per-shard leader 在 IDC 的場景）：

```bash
ansible -i inventory/crossregion.ini idc-dbhost-1 -m shell \
  -a 'systemctl stop {{ db_service }}'
```

## 預期行為

- **P-A**: IDC majority；leader node 死 → 剩 1 IDC voter + 1 GCP voter = 2 voter quorum；leader election → 新 leader 通常還在 IDC
- **P-B**: per-shard leader 散；停 IDC node ⇒ 只 shard-X-leader-IDC 失效；其他 shard leader (GCP) 不變

## 度量

| Metric | 來源 |
|---|---|
| Leader election RTO | timestamp 差：last write before stop → first write after election |
| tpmC dip + recovery curve | `chaos/C4/tpmc-1s.txt` |
| Per-DB election timeout 配置 | TiDB `raftstore.raft-election-timeout-ticks` / CRDB `kv.raft.election_timeout` / YBDB `raft_heartbeat_interval_ms` |

## 預期 RTO（各家 default）

| DB | Election timeout | 觀察 |
|---|---|---|
| TiDB | ~10s (raft-election-timeout = 10 ticks × 1s) | leader 切換 + region cache 刷新 |
| CRDB | ~9s | range lease re-election |
| YBDB | ~5-15s | raft heartbeat-based |

→ 對應 0602.md §10.4 表的 RTO 預估。

## Lab mode

- node stop 持續整 5 round（不重啟）
- 不測 split-brain（只測 leader election）

## Restore

```bash
ansible -i inventory/crossregion.ini idc-dbhost-1 -m shell \
  -a 'systemctl start {{ db_service }}'
```
