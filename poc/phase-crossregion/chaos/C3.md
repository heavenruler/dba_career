# Chaos C3 — GCP read-only (region quorum loss)

## 注入方式

不切 WAN，而是讓 GCP voter 三 node 失效（如 `systemctl stop` 三家 DB process），但 WAN 仍通。

```bash
ansible -i inventory/crossregion.ini gcp_db -m shell \
  -a 'systemctl stop {{ db_service }}'
```

## 預期行為

- **P-A**: GCP 3 voter 全死；每 shard 剩 2 IDC voter = quorum，**仍可寫**（IDC majority 保護）
- **P-B**: per-shard voter set = 1 IDC + 1 GCP + 1 arbiter；GCP voter 死 → 剩 1 IDC + 1 arbiter = 2 voter（still quorum if arbiter 在 IDC）；若 arbiter 在 GCP → 只剩 1 voter ⇒ quorum loss ⇒ 整 shard read-only

→ P-B placement 設計時 arbiter 位置決定 chaos C3 行為。

## 度量

| Metric | 來源 |
|---|---|
| tpmC drop curve | `chaos/C3/tpmc-1s.txt` |
| 全 cluster vs per-shard write fail rate | derived from go-tpc errors |
| Read availability | 應持平（read 可走 IDC voter）|

## Lab mode

- GCP 三 host stop 持續整 5 round
- 不測 process restart / recovery time

## Restore

```bash
ansible -i inventory/crossregion.ini gcp_db -m shell \
  -a 'systemctl start {{ db_service }}'
```
