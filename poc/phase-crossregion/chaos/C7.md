# Chaos C7 — cluster write reject (region majority loss)

## 注入方式

IDC 全 3 node 同時失效（極端情境）：

```bash
ansible -i inventory/crossregion.ini idc_db -m shell \
  -a 'systemctl stop {{ db_service }}'
```

## 預期行為

- **P-A**: IDC 3 voter 全死；每 shard 剩 1 GCP voter = minority ⇒ **全 cluster 寫拒**（split-brain 防護）
  - GCP follower 仍可 read（stale）
  - 應用層應收到 write rejection
- **P-B**: per-shard 1 IDC voter 死；剩 1 GCP voter + 1 arbiter；視 arbiter 位置決定 quorum

## 度量

| Metric | 來源 |
|---|---|
| Write failure rate | should be 100% after failure |
| Read availability | should remain (stale follower read) |
| GCP client retry storm | `chaos/C7/error-rate-by-sec.txt` |

## Lab mode

- 3 host stop 持續整 5 round
- 不測 recovery（不重啟 IDC nodes）
- 主測：`split-brain 防護` 是否啟動（無新 leader、無新 write commit）

## Validation

```sql
-- TiDB
SELECT * FROM information_schema.cluster_info;
-- 預期 IDC node status = down

-- CRDB
SELECT * FROM crdb_internal.kv_node_status;
-- 預期 IDC node alive = false

-- YBDB
yb-admin list_all_tablet_servers
-- 預期 IDC tserver = DEAD
```

## Restore

```bash
ansible -i inventory/crossregion.ini idc_db -m shell \
  -a 'systemctl start {{ db_service }}'
```
