# Chaos C1 — GCP partition (WAN drop, bi-directional)

## 注入方式

```bash
# IDC 側阻擋 GCP traffic
iptables -A INPUT  -s 10.160.152.0/24 -j DROP
iptables -A OUTPUT -d 10.160.152.0/24 -j DROP

# GCP 側阻擋 IDC traffic（對稱）
iptables -A INPUT  -s 172.24.0.0/16 -j DROP
iptables -A OUTPUT -d 172.24.0.0/16 -j DROP
```

## 預期行為

- **P-A**: IDC majority 仍可寫；GCP voter 變 minority；GCP client 走 idc-haproxy 仍 fail（路由斷）
- **P-B**: per-shard 兩區各佔 voter，partition 後**兩區皆 minority** ⇒ 全 cluster 寫拒（split-brain 防護生效）

## Pre-injection 檢查

```bash
ping -c 3 10.160.152.11   # 應通
nc -zv 10.160.152.13 26257  # CRDB port，應通
```

## Post-injection 預期 timeline

```
t=0   inject iptables drop
t+5s  raft heartbeat timeout
t+10s leader election kicks in (per-shard)
t+30s tpmC stabilises at degraded level
t+300s end of round
```

## Restore

```bash
iptables -D INPUT  -s 10.160.152.0/24 -j DROP
iptables -D OUTPUT -d 10.160.152.0/24 -j DROP
# (對稱於 GCP 側)
```
