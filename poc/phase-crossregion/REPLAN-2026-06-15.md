## §4 — chrony in-suite sampler + WAN probe

### Codex feedback applied
- iperf3 改 **opt-in only**（`WAN_PROBE_IPERF=1` 才開）；預設只 passive 採樣
- 改名「passive sampler」（移除「mrtg fallback」誤稱）
- iperf3 binary 依賴：先檢 `command -v iperf3`，缺則跳過該 sample
- 限定 `RESULT_SCOPE=X-CROSS && WAN_PROBE_ENABLED=1` 才啟動，**預設完全不改 S-BASE / S-K8S 流程**

### 計畫
- `phase-crossregion/scripts/wan-probe.sh`：
  - 預設模式（passive）：
    - chronyc tracking × 3 host pair（idc-driver↔gcp-poc-5 / idc-dbhost-1↔gcp-dbhost-1 / idc-haproxy↔gcp-haproxy），dump delta
    - `/proc/net/dev` byte counter delta（before/after round；passive）
  - opt-in 模式（`WAN_PROBE_IPERF=1`）：
    - iperf3 5s TCP test idc↔gcp（雙向）；只在 **round 間隙** 跑（warmup 結束 - sweep 開始之間，避免干擾 benchmark）
- `tests/common/run.sh` hook：
  - 限定 `RESULT_SCOPE=X-CROSS && WAN_PROBE_ENABLED=1` 才呼叫 wan-probe.sh
  - 採樣失敗 warn-only（不阻斷 sweep）
